Intro to Git

Why we need version control

  • People overwrite each others changes
  • Humans are bad at tracking file changes, computers are flawless
  • Version control lets you step back in time and see history

Intro to Git

How git solves this problem

  • Store final versions of files in one place (repository)
  • Track files locally, watch changes
  • Force local merge before pushing
  • This is only really useful for plain text (code, svg, documents)

Intro to Git

Git work flow

  1. pull changes from remote storage
  2. make changes to file
  3. review changes before
  4. push changes to remote storage

Intro to Git

Main git commands

  • git pull (get commits from remote)
  • git status (list modified files)
  • git diff [FILENAME] (Show differences between .git and files
  • git add FILE1 [FILE2 FILE3 ...] (add non-tracked files)
  • git commit -m"message!" (commit staged changes to .git)
  • git commit -am"message!" (commit all changes to .git)
  • git push (push committed changes to remote)
  • git stash (store changes temporarily)
  • git checkout FILENAME (undo changes since last commit
  • git checkout -b BRANCHNAME (create branch and switch to it)
  • git checkout BRANCHNAME (switch to branch)
  • git branch (list all local branches)
  • git merge BRANCHNAME (merge all changes of BRANCHNAME into current branch)
